# NTFS Raw Image and MFT Mirror Recovery - Plan

## Goal Capsule

| Field | Decision |
|---|---|
| Objective | Add a clean parser/core boundary for raw NTFS record evidence and `$MFTMirr` recovery so corrupted primary `$MFT` system records can be represented with explicit caveats instead of silent loss or ad hoc fallback. |
| Authority | This follows the completed NTFS physical-usage refactor and its deferred raw-image / `$MFTMirr` follow-ups. Fearless internal API changes are acceptable, but deletion behavior remains out of scope. |
| Execution profile | Parser-first Rust refactor with deterministic owned fixtures, then optional core/live integration once the parser contract is stable. |
| Stop condition | Stop for license-boundary risk, deletion-safety regression, or a need for private/live binary fixtures that cannot be represented with repo-owned generated bytes. |
| Landing strategy | Land in small conventional commits. Commit only source, tests, and docs; keep generated raw/live evidence under `target/`. |

## Product Contract

Rebecca's NTFS backend is now fast and physically descriptive, but its record source contract still treats the primary `$MFT` byte stream as the only parse authority. Mature NTFS tools do not do that blindly: the `$MFTMirr` metadata file can provide a bounded copy of the first system records when the primary table header records are damaged.

This plan adds that capability without turning Rebecca into a forensic image mounter. The parser should be able to merge primary and mirror record streams, prefer primary records when they parse, recover only records actually present in the mirror stream, and attach machine-readable caveats to recovered records. Core/live code can then reuse the same semantics for future raw-image fixtures and volume-level diagnostics.

### Requirements

- R1. Primary `$MFT` records remain authoritative when they parse successfully.
- R2. `$MFTMirr` recovery is bounded by the mirror bytes supplied by the caller; no code guesses extra records.
- R3. A recovered record receives a stable caveat such as `mft-mirror-record-used` that names the recovered record and the primary parse failure.
- R4. If both primary and mirror records fail, the primary parse error remains visible in `MftRecordBatch.errors`.
- R5. Truncated primary records can be recovered only when the matching mirror record is complete and valid.
- R6. Parser tests use generated repo-owned record bytes only.
- R7. Documentation explains that mirror recovery is read-only evidence and never deletion authority.

## Planning Contract

### Key Technical Decisions

- KTD1. Put mirror merge semantics in `rebecca-ntfs::MftRecordReader`.
  This keeps raw image, live volume, and future fixture callers from reimplementing NTFS parser policy in `rebecca-core`.
- KTD2. Keep `MftRecordBatch` shape stable for this slice.
  Recovered mirror records carry record-level caveats. Unrecovered primary failures remain batch errors, so existing core parse-error summarization continues to work.
- KTD3. Do not locate `$MFTMirr` in live volume code yet.
  The first deliverable is a deterministic parser contract. Live integration should follow once record-source code can supply mirror bytes without mixing Windows API details into parser policy.

### Scope Boundaries

- In scope: `rebecca-ntfs` mirror-aware reader APIs, deterministic tests, changelog, fixture documentation, and engineering memory.
- Out of scope: deleted-entry recovery, index slack parsing, write support, image mounting, broad forensic acquisition, and using mirror data to authorize cleanup deletion.
- License boundary: reference projects may inform behavior only. No external code, comments, layouts, or binary fixtures are copied.

## Implementation Units

### U1. Add mirror-aware reader merge

- **Goal:** Teach `MftRecordReader` to parse a primary MFT byte stream with an optional mirror byte stream.
- **Requirements:** R1, R2, R3, R4, R5, R6
- **Files:** `crates/rebecca-ntfs/src/reader.rs`, `crates/rebecca-ntfs/src/lib.rs`, `crates/rebecca-ntfs/tests/mft_parser.rs`
- **Approach:** Add `parse_records_with_mirror` and `parse_records_from_with_mirror`. Iterate the primary stream exactly as today. For each failed primary full record or truncated primary remainder, try the matching mirror record by record id. Use the mirror record only if it is complete and parses under strict fixup policy. Add `mft-mirror-record-used` to the recovered record. Keep existing `parse_records` behavior unchanged.
- **Test scenarios:**
  - A corrupt primary record 0 is recovered from mirror record 0 and includes the caveat.
  - A valid primary record wins over a different mirror record with the same id.
  - A truncated primary remainder is recovered when the mirror has the complete record.
  - A corrupt primary record remains an error when the mirror record is missing or invalid.
- **Verification:** `cargo nextest run -p rebecca-ntfs --test mft_parser`.

### U2. Document mirror recovery semantics

- **Goal:** Make mirror recovery understandable to maintainers and future raw-image work.
- **Requirements:** R6, R7
- **Files:** `CHANGELOG.md`, `crates/rebecca-ntfs/tests/fixtures/README.md`, `docs/knowledge/engineering/current-state.md`, `docs/knowledge/engineering/log.md`
- **Approach:** Add Unreleased changelog text, update fixture policy to mention generated mirror fixtures, and record the new parser boundary in engineering memory.
- **Test scenarios:**
  - Docs describe mirror recovery as read-only evidence.
  - Docs do not claim live volume `$MFTMirr` integration is complete.
- **Verification:** `git diff --check`.

## Definition of Done

- `MftRecordReader` can recover failed primary records from bounded mirror bytes.
- Recovered records have a stable `mft-mirror-record-used` caveat.
- Primary records still win when valid.
- Unrecovered primary parse failures remain visible as batch errors.
- Deterministic parser tests cover corrupt, valid-primary, truncated-primary, and invalid-mirror cases.
- Changelog, fixture docs, and engineering memory describe the capability and remaining live integration gap.
